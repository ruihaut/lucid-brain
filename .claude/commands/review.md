# /review — sprint review (GATE ③)

Verify the increment against the sprint goal and the vision, then update durable state.

## Steps

1. Verify the increment against the **sprint goal** and `VISION.md` (Goal-Driven verify loop: restate the success criteria, then check the result against each).
2. Demo or summarize what was delivered for the maintainer.
3. Update `scrum/backlog.md`: mark done items and add newly discovered items with a priority.
4. **Flush live state back to markdown** — the sprint file and backlog must reflect reality (durable-memory rule).
5. **GATE ③:** require explicit maintainer validation that the increment is accepted. If it is not accepted, capture what is missing as backlog items.
6. Hand off to `/retro`.
