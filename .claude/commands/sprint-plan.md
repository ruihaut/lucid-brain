# /sprint-plan — plan the next sprint

The DG (Product Owner function) and the directors plan a short, focused sprint. Markdown first, then mirror to the shared task list.

## Steps

1. Read `VISION.md` and `scrum/backlog.md`. Pull the top backlog items.
2. Define a **sprint goal**: a single slice of the finalité this sprint will deliver.
3. Write **INVEST stories** (Independent, Negotiable, Valuable, Estimable, Small, Testable) for that goal. For each: an owner (a director or specialist from `team/roster.md`) and a definition-of-done.
4. Write them to `scrum/sprints/sprint-NN.md` (next number). This markdown is authoritative.
5. **Then** seed the shared task list from those stories (Teams mode) so the team can coordinate. In Solo mode, note the stories the directors' subagents will execute.
6. Keep the sprint small (KISS) and aimed squarely at the goal (Goal-Driven Execution).
